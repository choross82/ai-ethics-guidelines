## 3. 1군: 거의 전 문서에 등장하는 핵심 키워드 (80% 이상)

### (1) 투명성 / 설명가능성 (Transparency / Explainability) — 18/19건

AI 시스템의 작동 원리, 의사결정 과정, 한계를 이해관계자에게 공개하고 설명할 수 있어야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| UNESCO | 투명성과 설명가능성 |
| OECD | 투명성과 설명가능성, 이의 제기 권리 |
| EU AI Act | 투명성, 기술 문서화 |
| NIST | 설명가능성 및 해석가능성 (Explainable and Interpretable) |
| 미국 AI 권리장전 | 통지 및 설명 (Notice and Explanation) |
| 바이든 EO | 콘텐츠 라벨링 및 출처 표시 |
| 블레츨리 선언 | 투명성 및 책임 |
| G7 히로시마 | 투명성 보고서 발간, 성능 한계 공개 |
| 한국 윤리기준 | 투명성, 설명가능성 |
| 중국 윤리규범 | 안전 투명성 |
| 싱가포르 GenAI | 콘텐츠 출처 투명성, 표준화된 정보공개 |
| 아실로마 원칙 | 실패 투명성 (피해 원인 파악), 사법 투명성 (감사 가능한 설명) |
| WHO 보건윤리 | 투명성, 설명가능성 및 이해가능성 |
| 유럽평의회 협약 | 투명성, AI임을 통지 |
| 일본 가이드라인 | 투명성, 검증가능성 |
| G20 원칙 | 투명성과 설명가능성 |
| IEEE 설계 | 투명성 (결정 근거의 발견가능성) |

### (2) 프라이버시 / 개인정보보호 (Privacy / Data Protection) — 18/19건

개인 데이터의 수집·저장·활용에 있어 정보주체의 권리를 보호하고, 프라이버시 침해를 방지해야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| UNESCO | 개인정보 보호, 데이터 거버넌스 |
| OECD | 데이터 신탁 |
| EU AI Act | 데이터 거버넌스, 데이터 품질 관리 |
| NIST | 프라이버시 강화 (Privacy-Enhanced) |
| 미국 AI 권리장전 | 데이터 프라이버시, 프라이버시 보호 기본 설계 (Privacy by Design) |
| 바이든 EO | 프라이버시와 시민 자유 보호, 프라이버시 강화 기술 (PETs) |
| G7 히로시마 | 프라이버시 보호, 개인정보 보호 |
| 한국 윤리기준 | 프라이버시 보호 |
| 한국 생성형AI | 개인정보 보호 (전체 문서의 핵심) |
| 중국 윤리규범 | 프라이버시 안전 보호, 개인정보 권리 |
| 싱가포르 GenAI | 개인정보 보호기술 (PETs) |
| 아실로마 원칙 | 개인 프라이버시, 자유와 프라이버시 |
| 몬트리올 선언 | 프라이버시 및 친밀성 보호, 디지털 단절의 권리 |
| WHO 보건윤리 | 프라이버시 보호, 데이터 보호법 |
| 유럽평의회 협약 | 프라이버시와 개인정보 보호 |
| 일본 가이드라인 | 프라이버시 보호, 개인정보 보호법 준수 |
| G20 원칙 | 프라이버시 |
| IEEE 설계 | 데이터 에이전시 (개인 데이터 접근/공유/통제) |

### (3) 안전성 (Safety) — 17/19건

AI 시스템이 인간의 생명·건강·재산에 해를 끼치지 않도록 설계·운영되어야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| OECD | 견고성, 보안, 안전성 |
| EU AI Act | 건강, 안전, 견고성 |
| NIST | 안전성 (Safe), 유효성 및 신뢰성 |
| 미국 AI 권리장전 | 안전하고 효과적인 시스템 |
| 바이든 EO | AI의 안전성, 레드팀 테스트 |
| 블레츨리 선언 | AI 안전 (AI Safety), 재앙적 피해 가능성 |
| G7 히로시마 | CBRN 위험, 자기복제 위험, 건강/안전 위험 |
| 한국 윤리기준 | 안전성 |
| 한국 생성형AI | 안전조치 |
| 중국 윤리규범 | 리스크 방범 (위험 예방) |
| 싱가포르 GenAI | 안전 모범사례, 안전 및 정렬 R&D |
| 아실로마 원칙 | 안전성, 재앙적/실존적 위험 |
| WHO 보건윤리 | 안전, 정확성, 효능성, 해악 방지 |
| 일본 가이드라인 | 안전성 (생명/신체/재산/정신/환경 위해 방지) |
| G20 원칙 | 견고성, 보안 및 안전 |
| IEEE 설계 | 안전, 안정성 |

### (4) 책임성 / 책무성 (Accountability / Responsibility) — 17/19건

AI 시스템의 개발·배포·운영에 관여하는 주체가 그 결과에 대해 책임을 져야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| UNESCO | 책임성 |
| OECD | 책임성, AI 행위자의 책임 |
| NIST | 책임성 및 투명성 (Accountable and Transparent) |
| 바이든 EO | 책임 있는 혁신, 이중용도 기반모델 보고 의무 |
| 블레츨리 선언 | 책임감 있는 (responsible) |
| G7 히로시마 | 책임 있는 정보 공유, 사고 보고 |
| 한국 윤리기준 | 책임성 (책임 주체 설정, 책임소재 명확화) |
| 한국 생성형AI | 개인정보 보호책임자(CPO) |
| 중국 윤리규범 | 책임 담당 강화, AI 문책 메커니즘 |
| 싱가포르 GenAI | 책임성 (Accountability), 공유 책임 모델 |
| 아실로마 원칙 | 책임 (설계자/구축자의 도덕적 책임) |
| 몬트리올 선언 | 책임 (AI 결정에 대한 인간 책임) |
| WHO 보건윤리 | 책임성 및 책무성, 인간 보증(human warranty) |
| 유럽평의회 협약 | 책임성 및 책무성, 구제 수단 (Remedy) |
| 일본 가이드라인 | 어카운터빌리티 (설명 책임) |
| G20 원칙 | 책무성 (Accountability) |
| IEEE 설계 | 책무성 (명확한 결정 근거 제공) |

### (5) 공정성 / 비차별 (Fairness / Non-discrimination) — 17/19건

AI 시스템이 인종·성별·연령·장애 등에 따른 차별적 결과를 낳지 않아야 하며, 편향을 최소화해야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| UNESCO | 다양성과 포용성, 소외 집단 보호 |
| OECD | 공정성, 사회정의, 불평등 감소 |
| EU AI Act | 사회적 점수화(소셜 스코어링) 금지, 차별적 AI 관행 금지 |
| NIST | 공정성 및 유해 편향 관리 |
| 미국 AI 권리장전 | 알고리즘 차별 보호, 대표성 있는 데이터, 알고리즘 영향평가 |
| 바이든 EO | 형평성과 시민권 증진, 차별과 편향 방지 |
| G7 히로시마 | 편향 완화, 아동/취약계층 보호 |
| 한국 윤리기준 | 다양성 존중, 편향과 차별 최소화, 사회적 약자의 접근성 보장 |
| 중국 윤리규범 | 공평공정 촉진, 편견 차별 회피 |
| 아실로마 원칙 | 문화적 다양성 |
| 몬트리올 선언 | 형평성, 다양성 포용, 성별/민족/문화/종교적 차별 방지 |
| WHO 보건윤리 | 포용성 및 형평성, 편향과 차별 방지 |
| 유럽평의회 협약 | 평등과 비차별, 장애인/아동 권리 특별 고려 |
| 일본 가이드라인 | 공평성, 불공정한 편향 배제 |
| G20 원칙 | 공평성, 비차별 |
| IEEE 설계 | 인권, 복지 |

### (6) 인간 중심 / 인간 존엄성 (Human-Centric / Human Dignity) — 16/19건

AI는 인간의 존엄성을 존중하고 인간의 이익을 위해 봉사해야 한다는 원칙.

| 문서 | 사용 표현 |
|------|---------|
| UNESCO | 인간 존엄성 존중 |
| OECD | 인간 중심 가치, 인간 역량 증강 |

| 블레츨리 선언 | 인간 중심적, 인간의 복지 |
| G7 히로시마 | 인권 보호 |
| 한국 윤리기준 | 인간성 (Humanity), 인간 존엄성 |
| 중국 윤리규범 | 인간 중심 원칙, 인간과 기계의 조화로운 공존 |
| 아실로마 원칙 | 인간적 가치 (존엄성, 권리, 자유) |
| WHO 보건윤리 | 자율성 보호, 인간의 복지 |
| 유럽평의회 협약 | 인간 존엄성, 개인 자율성 |
